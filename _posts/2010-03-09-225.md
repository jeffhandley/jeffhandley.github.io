<p>Do you have multiple personalities in Visual Studio?  I do.  And so do I.  I like to code with a dark background;I like to send people code snippets using more traditional colors.</p>  <p>Awhile back, I wrote about how to <a href="http://jeffhandley.com/archive/2009/03/31/vs-profiles.aspx" target="_blank">quickly change profiles in Visual Studio</a>, but with Visual Studio 2010, the approach changes slightly.  Here’s an update for how to toggle VS between different font/color themes, or other settings.</p>  <p>Here’s what we’re going to do:</p>  <ol>   <li>Write a macro to export the current settings, to allow for easy backups</li>    <li>Write a method to import all settings from a specified file</li>    <li>Write a macro to import your last settings backup file for easy restores</li>    <li>Manually export subsets of your settings for different purposes</li>    <li>Write macros to import each of the settings files exported</li>    <li>Integrate all of the above macros into the menu bar</li> </ol>  <p>Start off by clicking on Tools\Macros\Macros IDE.  Let’s create a new Macros Module for all of the “Settings” macros.  I chose to call it “Settings” of course.</p>  <p><img style="display: inline" title="Add New Item - MyMacros - Settings" alt="Add New Item - MyMacros - Settings" src="http://jeffhandley.com/images/jeffhandley_com/WindowsLiveWriter/QuicklyChangingYourVisualStudioTheme_C2C7/image_3.png" width="630" height="345" /> </p>  <p>For the first macro, let’s call it ExportActiveSettings() and it will export your current settings to a backup file, keeping a date-stamped backup file.  We’re just going to hard-code some strings into the macros; <p>
  <br />After saving this macro module, flip back over to Visual Studio.  Click on Tools\Macros\Macro Explorer.  Within that explorer, expand Settings and then double-click on “ExportActiveSettings.”  You might want to check the folder you used to make sure your active settings file was created.</p>

<p>Once you have saved this export, make changes to your settings and then export whatever other settings files you want.  One example is to change your fonts to a presentation mode where you use large fonts and few tool windows.  In the dialog for exporting settings, you can export the subsets of settings affected by your changes.  For my own use, I have creates vssettings files for the following:</p>

<ul>
  <li>Handley.Active.Colors.vssettings – uses my active settings, but the file only contains an export of my fonts/colors.  This would allow me to re-import my fonts/colors only, while leaving other settings alone.</li>

  <li>Default.Colors.vssettings – I reset the settings to the default color scheme and then exported the fonts/colors.  I’ll use this when sending code snippets to people in email, pasting into design documents, or using other media where my dark background theme doesn’t work as well.</li>

  <li>Presentation.vssettings – This uses my dark theme but large font sizes in the Text Editor, and I’ve made sure that all tools windows are auto-hidden.  I exported only the Options\Environment node when choosing settings to export.</li>

  <li>Formatting.Allman.vssettings – This has the C# code formatting set to the Allman style brackets and indentation settings.  I exported only these settings using the Options\Text Editor\C# Editor node when choosing settings to export.</li>

  <li>Formatting.KnR.vssettings – Just like Allman but with the settings switched to use KnR style code formatting.</li>
</ul>

<p>I ended up opening the Macro Explorer and double-clicking my “ImportActiveSettings” macro a couple of times during this process.</p>

<p>Notice that for ApplyAllmanFormatting and ApplyKnRFormatting, I went ahead and made the macro format the active document, because every time I’ve needed to switch between the 2 formats, I need to reformat the file I’m working on.</p>

<p>You’ll see that the Macro Explorer now shows all of the new macros you’ve added.  You can double-click on the macros and watch your settings get quickly imported without having to step through any dialogs.  <em><font color="#d26941">Be careful not to accidentally run ExportActiveSettings when you have goofy settings applied.</font></em></p>

<p>The only thing left to do now is to integrate these macros into your Visual Studio menu bar.  This is what ultimately makes these macros so useful.  As I mentioned on my older post on this subject, <a href="http://twitter.com/saraford" target="_blank">Sara Ford</a> <a href="http://blogs.msdn.com/saraford/archive/2008/12/05/did-you-know-you-can-create-toolbar-buttons-to-quickly-toggle-your-favorite-vs-settings-371.aspx" target="_blank">taught us</a> how to integrate macros into our menu/toolbar.  With Visual Studio 2010, I’ve found that we can no longer copy/paste/edit icons for our custom items, which is a bummer; instead we’ll just create a traditional menu for our needs.</p>

<ol>
  <li>Right-click on the menubar or toolbar in VS and choose “Customize…”</li>

  <li>Click on the Commands tab</li>

  <li>Select the “Menu bar” radio button and then select “Menu Bar” in the dropdown</li>

  <li>Scroll down and select the location where you’d like to insert your new menu, and click on “Add New Menu”</li>

  <ul>
    <li>You’ll get a new menu option called “New Menu”</li>
  </ul>

  <li>Click on Modify Selection and change the Name to something like “Settings”</li>

  <li>Up where “Menu Bar” is selected, choose “Settings” in the dropdown</li>

  <ul>
    <li>I find this rather unintuitive, but you get used to it</li>
  </ul>

  <li>Now, for each of the macros that you created, you’ll run the following steps:</li>

  <ol>
    <li>Click on “Add Command…”</li>

    <li>Select “Macros” as the category on the left</li>

    <li>Select the macro as the command on the right</li>

    <li>Click OK</li>

    <li>Click on Modify Selection and change the name to something concise</li>
  </ol>

  <li>When finished, click OK</li>
</ol>
